# Cloyee PRD (Product Requirements Document)
> 최초 작성: 2026.03 | 최종 업데이트: 2026.03.24 | 버전: v0.2

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | Cloyee |
| 배포 URL | https://cloyee.vercel.app |
| 한줄 소개 | AI 스터디메이트와 함께하는 나만의 성장 기록 공간 |
| 목적 | 학습 대화, 코드 리뷰, 기록을 통해 개발자의 성장을 정량화 |
| 타겟 사용자 | 스스로 성장을 측정하기 어려운 프론트엔드 경력 개발자 |

---

## 2. 배경 및 문제 정의

- 경력이 쌓여도 "내가 얼마나 성장했는지" 체감하기 어렵다
- 혼자 공부하면 방향을 잡기 어렵고 지속하기 힘들다
- 코드 리뷰를 받을 기회가 적다
- 기획, 디자인, AI 활용 등 개발 외 영역도 학습이 필요하지만 체계가 없다

---

## 3. 핵심 가치

> "Cloyee는 단순한 학습 앱이 아니라, 나의 성장을 함께 기록하는 AI 스터디메이트다"

- **대화형 학습** — 일방적인 강의가 아닌 Cloyee와의 대화로 이해도 확인
- **즉각적인 피드백** — 코드 작성 후 바로 리뷰 가능
- **성장의 가시화** — 쌓인 학습이 레벨과 기록으로 보임
- **카테고리 기반** — 개발에 국한되지 않는 전방위 성장

---

## 4. 사용자 (페르소나)

**메인 페르소나: 49세 프론트엔드 개발자**

- 퍼블리싱 + 프론트엔드 경력 18년
- HTML/CSS, JS, React 실무 경험 보유
- 스스로 실력을 과소평가하는 경향
- AI 툴에 관심 있고 새로운 방향을 모색 중
- 체계적인 학습보다 대화형 학습을 선호

## 5. 핵심 기능

### 5-1. 카테고리 선택 ✅ 완료

| 항목 | 내용 |
|------|------|
| 기능 | 학습할 카테고리를 선택 |
| 기본 카테고리 | 개발 / 기획 / 디자인 / AI 활용 |
| 추후 기능 | 사용자가 직접 카테고리 추가 (v0.2 P1, DB 준비 완료) |

---

### 5-2. 학습 대화 ✅ 완료

| 항목 | 내용 |
|------|------|
| 기능 | Cloyee와 대화하며 개념 학습 및 이해도 확인 |
| 방식 | 대화형(직접 입력) / 선택형(4지선다) 분기 |
| 특징 | 소크라테스식 대화, SSE 스트리밍 실시간 출력 |
| 진입 분기 | 첫학습 / 이어서(복습 or 바로) 자동 판단 |
| 진도율 | 문항 수 기반 ("N문항 학습 중"), 완료 시 CompleteBanner |

---

### 5-3. 코드 리뷰 ✅ 완료

| 항목 | 내용 |
|------|------|
| 기능 | 작성한 코드를 입력하면 Cloyee가 리뷰 제공 |
| 리뷰 항목 | 코드 품질, 개선점, 대안 방법 제안 |
