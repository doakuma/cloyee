# Cloyee — 기능 요구사항 (MoSCoW)

> 기능의 우선순위를 Must / Should / Could / Won't 로 분류합니다.
> 각 버전별로 범위를 명확히 하고, 스코프 크리프를 방지합니다.

---

## 📌 v0.1 — MVP (완료)

### Must ✅
- [x] 카테고리 선택 → 학습(chat) / 코드리뷰(review) 플로우
- [x] Socratic 대화 (AI가 질문으로 유도)
- [x] 이해도 점수 (1~5)
- [x] 학습 세션 자동 저장
- [x] 성장 시각화 (레벨, 경험치, 스트릭)

### Should ✅
- [x] 모바일 반응형
- [x] 다크/라이트 모드
- [x] 세션 히스토리 조회

### Could ✅
- [x] 코드 리뷰 모드
- [x] 여러 카테고리 지원

### Won't
- 오프라인 모드
- 실시간 협업
- 음성 인식

## 🚫 명시적으로 하지 않을 것 (Won't Do)

| 항목 | 이유 |
|------|------|
| TypeScript 도입 | 개발 속도 > 타입 안정성 (현 규모) |
| 오프라인 모드 | 인터넷 필수 서비스 (AI 학습) |
| 실시간 협업 | 1인 학습 앱이 기본 목표 |
| 음성/영상 | 모바일 웹에서 복잡도 증가 |
| 블록체인/NFT | 학습 앱에 불필요 |
