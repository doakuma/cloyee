# Cloyee — AI 토큰 비용 및 최적화 전략

> 작성일: 2026.03.16

---

## 현재 구조 (토큰 소비 흐름)

사용자가 문항 하나에 답할 때마다 `/api/chat` API 호출 1회 발생.

1회 호출 시 소비 토큰:

- 시스템 프롬프트: 매 호출마다 전체 전송 (사용자 프로필 + 로드맵 정보 + chatMode 지침 포함, 약 1,000~2,000 토큰)
- 대화 히스토리: 누적 전송 (대화가 길어질수록 선형 증가)
- AI 응답: 1회 생성 (약 300~600 토큰)

---

## 예상 비용 (claude-haiku-4-5 기준)

| 단위                      | 토큰 소비         | 비용 (약)     |
| ------------------------- | ----------------- | ------------- |
| 세션 1회 (문항 10개)      | 5,000~15,000 토큰 | $0.001~$0.003 |
| 사용자 100명 × 하루 1세션 | —                 | 월 $3~$9      |

현재 베타 단계에서는 무시할 수 있는 수준.
프로덕션에서 claude-sonnet-4-6 전환 시 비용 3~5배 증가 예상.

## 구조적 문제 (스케일 시 이슈)

### 문제 1 — 시스템 프롬프트 비대화

- 매 호출마다 동일한 시스템 프롬프트 전체 전송
- 사용자 프로필, 로드맵 정보, chatMode 지침 등 누적
- 절감 가능 토큰: 호출당 500~800 토큰

### 문제 2 — 히스토리 무제한 누적

- 대화 전체를 매 호출마다 전송
- 20턴 대화 시 초반 내용이 매번 전달됨
- 절감 가능 토큰: 긴 대화에서 50~70%

## 최적화 방안 (우선순위)

| 우선순위 | 방법                                                                  | 효과                     | 난이도 |
| -------- | --------------------------------------------------------------------- | ------------------------ | ------ |
| P1       | 시스템 프롬프트 경량화 — 불필요한 지침 제거, 핵심만 유지              | 호출당 500~800 토큰 절감 | 낮음   |
| P2       | 히스토리 슬라이싱 — 최근 N턴(예: 6턴)만 전송                          | 긴 대화에서 50~70% 절감  | 낮음   |
| P3       | 프로덕션 모델 전환 시 구간 설계 — 짧은 대화는 haiku, 긴 대화는 sonnet | 비용 제어                | 중간   |

## 현재 결정사항

- 베타 단계: 현재 구조 유지 (비용 무시 가능 수준)
- 사용자 증가 전 P2(히스토리 슬라이싱) 먼저 적용 권장
- 프로덕션 전환 시 시스템 프롬프트 경량화(P1) 병행

## P2 히스토리 슬라이싱 구현 메모

```js
// src/app/api/chat/route.js
// 전체 히스토리 대신 최근 6턴만 전송
const recentMessages = (messages ?? [])
  .map(({ role, content }) => ({ role, content }))
  .slice(-6); // 최근 6개만

const conversationMessages = [
  ...recentMessages,
  { role: "user", content: message },
];
```

주의: 너무 짧게 자르면 AI가 이전 맥락을 잃음. 6~10턴이 적절.
